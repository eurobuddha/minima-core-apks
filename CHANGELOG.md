- 2026-06-27 · minima-history 0.1.2 — Fix: empty list on a node with lots of history. The node caps any command response at 256,000 bytes ('results too long'); on a contract-heavy node even a 25-txpow history page blows past it, the node drops the reply, and the app got nothing. Sync is now ADAPTIVE: it starts at max:8 and, on any over-limit/empty/errored page, halves the page (8→4→2→1) and retries the same offset, keeping the smaller size — so it converges to a page that fits under 256KB and pulls the history a few txpows at a time. One-time backfill (pages to the end, resumable) vs steady-state incremental (stops at the first already-stored txpow). A single txpow that alone exceeds 256KB is skipped so it can't stall the sync. The status line now shows sync progress / a clear failure hint instead of a silent blank. Still bounded + gentle (delays, fetch cap).
