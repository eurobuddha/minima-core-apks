- 2026-06-24 · minima-utxo-wallet 0.3.1 — Reduce node load to keep Minima Core stable until its IPC is hardened. The per-reload command set drops from 6 cmds/~28KB to 4 cmds/~1.4KB: 'scripts' (~27KB, the wallet's stable 64-address pool) is now fetched once on load then only every ~20 blocks instead of every reload; 'getaddress' moved off the reload loop to a lazy fetch when the Receive tab is opened (result cached so display + Copy both work — also fixes a latent copy bug); reload debounce raised 200ms->400ms to coalesce the NEWBLOCK+NEWBALANCE burst into one reload. block/coins/balance still fetched every block (spend decisions + Distribute need fresh coins). No behaviour change to History/Wallet/Send
- 2026-06-24 · minima-utxo-wallet 0.3.0 — History rebuilt from scratch, node-friendly (lessons from myHistory). Shows SENT + RECEIVED + SELF, classified client-side from the node's history. The heavy 'history' command is now called ONLY while the History tab is visible and at most once per new block (this every-reload hammering is what crashed Minima Core) — bounded offset pagination (max:25 per call, ~350KB, well under the IPC limit; 'Load older' pages via offset, never a giant call). Lazy detail on tap; in-flight sends merged on top and auto-confirmed with the real on-chain txpowid + working explorer link by matching spent inputs; stale sends reconciled; Re-post/Dismiss on failed. TxResolver removed. Includes all code-review fixes (pagination, stale-pending fallback, expand-state stability, onResume refetch, multi-token hint)
- 2026-06-24 · minima-utxo-wallet 0.2.2 — HOTFIX: stop crashing Minima Core. Root cause (diagnosed via 3 explore agents + live node): 0.2.1's background txpowid resolver called the node's heavy 'history' command on EVERY reload (every block/balance event) forever, hammering the node's single-threaded IPC receiver until it crashed. Removed that every-reload call entirely. Also made History row details build lazily on tap instead of eagerly (no more main-thread view explosion). Full History rebuild (sent+received, node-friendly) ships next in 0.3.0
